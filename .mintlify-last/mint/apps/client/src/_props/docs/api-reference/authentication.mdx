# API Authentication

This guide explains how to authenticate your requests to the dstack API.

## Authentication Methods

dstack supports several authentication methods:

1. **API Keys** (recommended)
2. **OAuth 2.0**
3. **JWT Tokens**

## API Keys

API keys are the simplest way to authenticate with the dstack API.

### Generating an API Key

1. Log in to the dstack dashboard
2. Navigate to Settings > API Keys
3. Click "Create New API Key"
4. Give your key a name and select the appropriate permissions
5. Copy the API key (it will only be shown once)

### Using API Keys in Requests

Include your API key in the `Authorization` header:

```bash
curl -X GET "https://api.dstack.com/v1/projects" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## OAuth 2.0

dstack supports OAuth 2.0 for more secure authentication flows.

### Authorization Code Flow

1. Redirect users to:

```
https://auth.dstack.com/oauth/authorize?client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_REDIRECT_URI&response_type=code&scope=read,write
```

2. Exchange the code for an access token:

```bash
curl -X POST "https://auth.dstack.com/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code&code=AUTHORIZATION_CODE&client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET&redirect_uri=YOUR_REDIRECT_URI"
```

## JWT Tokens

For service-to-service communication, dstack supports JWT tokens.

### Generating a JWT Token

```javascript
const jwt = require('jsonwebtoken');

const token = jwt.sign(
  { 
    sub: 'service-account-name',
    iss: 'your-issuer',
    aud: 'dstack-api'
  },
  'your-secret-key',
  { expiresIn: '1h' }
);
```

### Using JWT Tokens in Requests

```bash
curl -X GET "https://api.dstack.com/v1/projects" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Best Practices

* Never expose your API keys or client secrets in client-side code
* Use appropriate scopes to limit access
* Rotate your API keys periodically
* Use environment variables to store sensitive credentials in your applications
