# API Reference Overview

dstack provides a comprehensive API that allows you to programmatically interact with all aspects of the platform.

## API Basics

### Base URL

All API requests are made to the following base URL:

```
https://api.dstack.com/v1
```

### Response Format

All responses are returned in JSON format. Here's a typical response structure:

```json
{
  "data": {
    // Response data here
  },
  "meta": {
    "status": 200,
    "requestId": "req_1234567890"
  }
}
```

### Error Handling

Error responses follow this format:

```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable error message",
    "details": {
      // Additional error details
    }
  },
  "meta": {
    "status": 400,
    "requestId": "req_1234567890"
  }
}
```

## Rate Limiting

The API implements rate limiting to ensure fair usage:

- **Free tier**: 60 requests per minute
- **Pro tier**: 300 requests per minute
- **Enterprise tier**: Custom limits available

## Available Endpoints

The API is organized into the following resource groups:

- **Projects**: Manage projects
- **Environments**: Configure environments
- **Deployments**: Control deployments
- **Resources**: Manage infrastructure resources
- **Users**: User management
- **Teams**: Team management

## Authentication

To use the API, you need to authenticate your requests. Learn more on the [Authentication page](/api-reference/authentication).

## SDKs

dstack provides official SDKs for the following languages:

- JavaScript/TypeScript
- Python
- Go
- Java
