# Overview: Supplementary Security Modules in dstack

This section provides a high-level summary of the supplementary security modules that are part of the broader dstack repository. These modules—**Certbot Client**, **Certificate Transparency Monitor**, **Host API Security**, and **IOHash Security**—are maintained as distinct code repositories, but are designed to work together to enhance the overall security posture of dstack's TEE (Trusted Execution Environment) infrastructure.

Each module addresses a specific aspect of security:

- **Certbot Client**: Automates certificate management and ACME protocol operations.
- **Certificate Transparency Monitor**: Monitors certificate issuance and detects anomalies.
- **Host API Security**: Secures host management and orchestration APIs.
- **IOHash Security**: Provides content-addressed storage and cryptographic integrity verification.

These systems are deeply integrated into dstack's architecture, and understanding their roles will help you appreciate the layered security model employed throughout the platform.
