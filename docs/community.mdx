## Contributing to dstack

We welcome contributions from the community! Here's how you can help:

### Code Contributions

1. **Fork the Repository**
   ```bash
   git clone https://github.com/Dstack-TEE/dstack.git
   cd dstack
   ```

2. **Create a Feature Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make Your Changes**
   - Follow the coding style guidelines
   - Add tests for new functionality
   - Update documentation as needed

4. **Submit a Pull Request**
   - Provide a clear description of your changes
   - Reference any related issues
   - Ensure all tests pass

### Documentation Contributions

Help improve our documentation:
- Fix typos or clarify explanations
- Add examples or use cases
- Translate documentation
- Create tutorials

### Testing and Feedback

- Test new releases and report issues
- Provide feedback on user experience
- Share your use cases and success stories

```bash
# Clone the repository
git clone https://github.com/Dstack-TEE/dstack.git

# Install dependencies
```

## FAQ

### General Questions

**Q: What is dstack?**  
A: dstack is an open-source platform for deploying secure applications to Trusted Execution Environments (TEEs).

**Q: Which TEE hardware is supported?**  
A: Currently, Intel TDX is the primary supported platform, with plans for broader hardware support.

**Q: Is dstack production-ready?**  
A: dstack is actively developed and used in production environments. Always test thoroughly for your use case.

### Technical Questions

**Q: Can I use dstack without TEE hardware?**  
A: You can develop and test locally, but full security features require TEE hardware. Try [Phala Cloud](https://cloud.phala.network) for testing without hardware.

**Q: How does dstack compare to other TEE platforms?**  
A: dstack focuses on developer experience, using familiar tools like Docker while providing enterprise-grade security.

**Q: What programming languages are supported?**  
A: Any language that can run in a Docker container. We provide SDKs for Go, Python, and JavaScript.
