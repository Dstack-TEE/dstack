## Step-by-Step Quick Deployment

### Prerequisites
- Docker installed locally (for building images)
- Basic command line knowledge
- Your application containerized (we'll help if it isn't)

### Step 1: Get Phala Cloud Access

First, obtain access to Phala Cloud and set up your development environment:

<details>
<summary>Environment Setup Commands</summary>

```bash
# Install Node.js and required tools
# (Skip if you already have these)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18.18.0
nvm use 18.18.0

# Initialize your project directory
mkdir my-secure-app && cd my-secure-app
yarn init -y
yarn install
```

</details>

1. **Register for Phala Cloud**: Visit [cloud.phala.network](https://cloud.phala.network/) and create your account
2. **Claim free credits**: Get started with complimentary TEE compute credits  
3. **Download CLI tools**: Install the Phala CLI for streamlined deployments

### Step 2: Prepare Your Application  

If you already have a Docker container, skip to Step 3. Otherwise, let's containerize a sample application:

<details>
<summary>Sample Application (Python Flask)</summary>

Create a simple secure web application:

```python
# app.py
from flask import Flask, jsonify
import os

app = Flask(__name__)

@app.route('/')
def secure_hello():
    return jsonify({
        'message': 'Hello from Phala TEE!',
        'secure': True,
        'tee_environment': os.environ.get('PHALA_TEE', 'enabled'),
        'deployment': 'phala-cloud'
    })

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)
```

```dockerfile
# Dockerfile
FROM python:3.11-slim
WORKDIR /app
RUN pip install flask
COPY app.py .
EXPOSE 8080
CMD ["python", "app.py"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "8080:8080"
    environment:
      - PHALA_TEE=enabled
```

</details>

### Step 3: Deploy to Phala Cloud

Once your application is containerized, deploy it to secure TEE infrastructure:

<details>
<summary>Deployment Commands</summary>

```bash
# Authenticate with Phala Cloud
phala auth login <YOUR_TOKEN>

# Build and push your container  
docker build -t my-secure-app .
docker tag my-secure-app <your-registry>/my-secure-app:latest
docker push <your-registry>/my-secure-app:latest

# Deploy to Phala TEE infrastructure
phala cvms create \
  --name my-first-secure-app \
  --compose docker-compose.yml \
  --skip-env
```

</details>

That's it! Your application is now running in a hardware-secured Trusted Execution Environment.

### Step 4: Verify Security

Access your deployed application and verify its security properties:

1. **Test functionality**: Your app will be accessible via a secure HTTPS endpoint
2. **Verify TEE attestation**: Use built-in attestation endpoints to cryptographically prove security
3. **Monitor deployment**: View logs, metrics, and security status through the Phala Cloud dashboard

## Why This Path Works

This super quick start demonstrates dstack's core value: **existing applications gain enterprise-grade security without code changes**. Your Docker containers run in hardware-isolated environments with:

- **Memory encryption**: All data encrypted in CPU cache and RAM
- **Remote attestation**: Cryptographic proof your code runs unmodified  
- **Zero-trust compute**: Even cloud admins cannot access your data
- **Instant deployment**: Production security in development timeframes

<Accordion title="Automated Shell Deployment Script">
  <div className="space-y-4">
    <p className="text-sm text-gray-700">This comprehensive script automates the entire deployment process from environment setup through application deployment:</p>
    
    <CodeBlock language="bash">
{`#!/bin/bash
# Automated dstack deployment to Phala Cloud

# Step 1: Environment Setup
echo "Setting up development environment..."
yarn init -y
nvm use 18.18.0
yarn install

# Step 2: Download TEE Simulator (for local testing)
echo "Downloading TEE simulator for local development..."
curl -L -O https://github.com/Leecheal/tappd-simulator/releases/download/v0.1.4/tappd-simulator-0.1.4-aarch64-apple-darwin.tgz
tar -xvf tappd-simulator-0.1.4-aarch64-apple-darwin.tgz
chmod +x run_simulator.sh
./run_simulator.sh &

# Step 3: Authenticate with Phala Cloud
echo "Authenticating with Phala Cloud..."
echo "Replace <TOKEN> with your actual Phala Cloud authentication token"
read -p "Enter your Phala Cloud token: " PHALA_TOKEN
phala auth login $PHALA_TOKEN

# Step 4: Build and Deploy Application
echo "Building Docker image..."
echo "Ensure your Dockerfile and docker-compose.yml exist in current directory"
read -p "Enter your image name (e.g., username/myapp): " IMAGE_NAME

docker build -t $IMAGE_NAME .
docker push $IMAGE_NAME:latest

# Step 5: Multi-platform Build (optional for broader compatibility)
echo "Building multi-platform image for better compatibility..."
docker buildx build --platform linux/amd64,linux/arm64 -t $IMAGE_NAME:latest --push --provenance=false .

# Step 6: Update Docker Compose Configuration
echo "Updating docker-compose.yml with proper port mapping..."
chmod +x update_compose.sh
./update_compose.sh

# Step 7: Deploy to Phala Cloud
read -p "Enter CVM name for deployment: " CVM_NAME
phala cvms create --name $CVM_NAME --compose docker-compose.yml --skip-env

echo "Deployment complete! Check Phala Cloud dashboard for status."
`}
    </CodeBlock>
    
    <div className="bg-blue-50 p-3 rounded">
      <p className="text-xs text-blue-800"><strong>Usage:</strong> Save as deploy.sh, make executable with chmod +x deploy.sh, then run ./deploy.sh</p>
    </div>
  </div>
</Accordion>

<Accordion title="Docker Compose Configuration Helper Script">
  <div className="space-y-4">
    <p className="text-sm text-gray-700">This helper script generates and configures your docker-compose.yml file for optimal Phala Cloud deployment:</p>
    
    <CodeBlock language="bash">
{`#!/bin/bash
# update_compose.sh - Docker Compose Configuration for Phala Cloud

set -e

echo "========================================================"
echo "Configuring Docker Compose for Phala Cloud Deployment"
echo "========================================================"

# Create backup of existing configuration
if [ -f "docker-compose.yml" ]; then
  echo "Creating backup of existing docker-compose.yml..."
  cp docker-compose.yml docker-compose.yml.bak
  echo "Backup saved as docker-compose.yml.bak"
fi

# Generate base docker-compose.yml using Phala CLI
echo "Generating new docker-compose.yml configuration..."
phala docker generate -o docker-compose.yml

# Verify generation was successful
if [ $? -ne 0 ]; then
  echo "❌ Error: Failed to generate docker-compose.yml"
  echo "Please check your Phala CLI installation and authentication"
  exit 1
fi

# Configure port mapping and environment variables
echo "Adding port mapping and optimizing configuration..."
TMP_FILE=$(mktemp)

# Process the generated file to add proper port mapping
cat docker-compose.yml | awk '
  {
    if ($0 ~ /container_name: app/) {
      print $0;
      print "    ports:";
      print "      - \"8080:8080\"";
      next;
    } else if ($0 ~ /- PORT=\${PORT}/) {
      # Remove automatic PORT environment variable (handled by port mapping)
      next;
    } else {
      print $0;
    }
  }
' > "$TMP_FILE"

mv "$TMP_FILE" docker-compose.yml

echo "✅ Docker Compose configuration completed successfully"
echo "You can now deploy your CVM with: phala cvms create --name <cvm-name> --compose docker-compose.yml"
echo "========================================================"

# Display final configuration for review
echo "Final docker-compose.yml configuration:"
cat docker-compose.yml
`}
    </CodeBlock>
    
    <div className="bg-green-50 p-3 rounded">
      <p className="text-xs text-green-800"><strong>Features:</strong> Automatic backup, error handling, port optimization, and configuration review</p>
    </div>
  </div>
</Accordion>
